## 1. 合规与法规 (Compliance)

## 4. 安全红线 (Security)

### 4.1 支付安全

| 威胁 | 防护措施 |
|:---|:---|
| **交易欺诈** | 实时风控、设备指纹 |
| **中间人攻击** | 证书固定、端到端加密 |
| **重放攻击** | 时间戳、随机数、签名验证 |

### 4.2 数据安全

```
敏感数据加密层级:
- 传输层: TLS 1.3
- 应用层: AES-256-GCM
- 存储层: 数据库字段级加密 + 密钥管理服务 (KMS)
```

### 4.3 反洗钱 (AML)

- **大额交易监控**: 单笔 > 5万人民币
- **频繁交易监控**: 24小时内 > 10笔
- **异常地区监控**: 高风险地区交易
- **黑名单过滤**: OFAC、联合国制裁名单

## 5. 金融业务特定考虑

### 5.1 核心业务模式

| 模式 | 关键指标 | 合规要点 |
|:---|:---|:---|
| **支付** | 成功率、费率 | 支付牌照、备付金存管 |
| **借贷** | 坏账率、利率 | 征信授权、利率披露 |
| **理财** | 申购赎回、收益率 | 投资者适当性、风险揭示 |
| **保险** | 理赔率、保费 | 精算合规、销售资质 |

### 5.2 风控指标

```python
# 伪代码示例
def assess_transaction_risk(transaction):
    risk_score = 0

    # 大额交易
    if transaction.amount > 50000:
        risk_score += 30

    # 异常时间
    if transaction.hour in [2, 3, 4]:
        risk_score += 20

    # 新用户
    if transaction.user.account_age_days < 7:
        risk_score += 25

    # 异常设备
    if transaction.device != transaction.user.usual_device:
        risk_score += 15

    return risk_score
```

## 7. 常见陷阱 (PITFALLS)

### 7.1 技术陷阱

- ❌ **不使用 HTTPS**: 金融应用必须全站 HTTPS
- ❌ **前端存储敏感数据**: 永远不要在前端存储完整卡号
- ❌ **忽略日志脱敏**: 日志必须脱敏处理
- ❌ **硬编码密钥**: 使用密钥管理服务 (KMS)

### 7.2 合规陷阱

- ❌ **未告知即收集数据**: 违反 PIPL
- ❌ **超范围使用数据**: 用户授权范围外使用
- ❌ **忽略用户注销权**: 必须提供注销账户功能
- ❌ **跨境数据传输**: 需安全评估和审批
