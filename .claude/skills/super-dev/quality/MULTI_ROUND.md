# 三轮质量审查协议 (Multi-Round Review)

> 每份输出必须经过三轮审查，禁止一次定稿。

## 流程概览

```
输入需求
    ↓
[Round 1] 生成 + 自检
    ↓
[Round 2] 批判 + 改进
    ↓
[Round 3] 终审 + 定稿
    ↓
商业级输出
```

---

## Round 1: 生成 + 自检

**目标**：生成完整初始版本，进行第一轮自检。

**步骤**：
1. 使用对应专家的知识和模板
2. 覆盖所有必要章节
3. 嵌入研究洞察
4. 使用 CHECKLIST.md 对应清单自检
5. 记录不足项

**退出条件**：通过率 >= 60%

---

## Round 2: 批判 + 改进

**目标**：以苛刻评审者视角批判初稿，针对性改进。

**批判维度**：
| 维度 | 批判问题 |
|:---|:---|
| 完整性 | 有没有遗漏重要内容？ |
| 准确性 | 数据是否有来源？ |
| 清晰性 | 读者能一次理解吗？ |
| 可执行 | 团队能直接按照执行吗？ |
| 逻辑性 | 有无前后矛盾？ |
| 深度 | 是否只是表面？ |
| 专业性 | 是否体现领域专业知识？ |

**退出条件**：
- 所有严重问题已修复
- 改进后通过率 >= 80%

---

## Round 3: 终审 + 定稿

**目标**：最终质量确认，添加质量保证声明。

**最终检查**：
1. 所有严重问题已修复
2. 文档结构完整
3. 术语一致
4. 无拼写/语法错误
5. 图表清晰可读
6. 版本/日期正确

**质量保证声明**：

```markdown
---
## 质量保证声明

本文档经过以下质量流程：
- 深度发现: 收集了 X/25 个关键问题
- 专业研究: 执行了 X 次联网搜索
- Round 1 自检: 通过率 X%
- Round 2 批判: 修复 X 个问题
- Round 3 终审: 通过率 X%

生成日期: [日期]
版本: v1.0
```

---

## 质量红线

以下情况**不允许定稿**：
- 任何严重问题未修复
- Round 3 检查通过率 < 100%
- 无质量保证声明
- 检查清单总通过率 < 80%
