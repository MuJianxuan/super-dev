# 自动研究与推断协议 (Research & Inference Protocol)

> **原则**: 用户不说的，我们就去查；查不到的，我们就用最佳实践补全。

## 1. 对标产品分析 (Benchmarking)

**触发条件**: 发现阶段识别出对标产品 (如 "QQ音乐")

**执行动作**:
1. **自动研究**: 搜索对标产品的核心功能、用户评价、定价策略。
   - *Query*: "QQ音乐 core features list", "QQ音乐 user complaints", "QQ音乐 pricing model"
2. **功能映射**: 将对标产品的核心功能作为我们产品的 "Default Scope"。
3. **差异化标记**: 标记出用户明确要求的不同点。

---

## 2. 行业标准补全 (Industry Standards)

对于用户未明确的细节，**禁止**反复询问，直接采用行业标准：

| 领域 | 缺失信息 | 自动补全策略 |
|:---|:---|:---|
| **认证 (Auth)** | 登录方式 | 默认: 手机号 + 验证码 + 社交登录 (微信/Apple) |
| **用户 (User)** | 个人中心 | 默认: 头像、昵称、设置、我的订单/收藏 |
| **合规 (Compliance)** | 协议 | 默认: 用户协议、隐私政策、GDPR (如涉及海外) |
| **UI** | 设计风格 | 默认: 平台原生风格 (iOS Human Interface / Material Design) |
| **异常 (Error)** | 异常流程 | 默认: 断网、超时、空状态、服务器错误 |
| **管理 (Admin)** | 后台管理 | 默认: 用户管理、内容管理、数据统计 |

---

## 3. 专业领域研究 (Domain Research)

**触发条件**: 涉及特定行业 (如 "医疗"、"金融")

**必须获取**:
1. **合规要求**: "Healthcare app compliance checklist"
2. **数据标准**: "FHIR standard overview"
3. **行业基准**: "Telemedicine app retention benchmarks"

---

## 4. 研究输出格式 (Inferred Specs)

在生成最终文档前，先整理一份 "推断摘要"：

```markdown
## 智能规划摘要 (基于 QQ音乐 对标)

### ✅ 已确认 (来自用户)
- 核心差异: 高保真音质 + 区块链版权
- 目标用户: 35+ 发烧友
- 平台: iOS Only

### 🤖 自动规划 (来自对标 & 推断)
- **基础功能**: 播放器、歌单、搜索、排行榜 (复用 QQ音乐 标准)
- **登录体系**: 手机号一键登录
- **设计风格**: 深色模式 (符合发烧友偏好)
- **合规性**: 内置数字版权保护 (DRM)
```
